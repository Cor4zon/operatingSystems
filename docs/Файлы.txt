Файлы.txt

[ЧТО?]
Файл - логический информационный блок, который создается процессами.

Файл - механизм абстрагирования. 
Файл представляет собой способ сохранения информации на диске и последующего ее считывания, который должен оградить пользователя от подробностей о способе и месте хранения информации и деталей фактической работе дисковых устройств.

Файл можно рассматривать, как разновидность адресного пространства, за исключением того, что файлы используются для моделирования диска, а не оперативной памяти.


[wiki] 
Файл - это специально организованная структура данных, распознаваемая компьютером как единое целое.

[КАК УСТРОЕННО?]
На диске содержаться тысячи и миллионы файлов.

Файлами рулит операционная система.



про расширения:
UNIX пофиг на расширение файла, расширение нужно юзеру, как напоминание и нужно некоторым программ (например, компилятор gcc примет только .c файл)


Windows не пофиг. Винда присваивает каждому расширению какое-то значение и по этому значению запускает прогу (например .doc -> Microsoft Documents Office ...)


[КАК УСТРОЕНО?]
Структура файла

Файлы могут быть структурированны разными способами.
Три самых популярных:
1. бессистемная последовательность байт;
(максимум гибкости - положить можно все, что угодно)

2. последовательность структур одинаковой длинны, каждая запись имеет структуру. 
(по ходу устарела)
операция чтения - возвращает 1 запись
операция записи - редактирует 1 запись

3. дерево (быстрый поиск)
используется в больших системах при обработке коммерч данных
